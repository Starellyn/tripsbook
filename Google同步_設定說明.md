# 記帳頁 → Google 試算表　同步設定說明

讓記帳頁／分帳頁的每一筆，自動寫進你的 Google 試算表（永久保存、手機電腦都能看、可多人查看）。設定一次，約 20–30 分鐘。

> 運作架構：
> ```
> 記帳頁（GitHub Pages）→ Apps Script 網址（你發布的小後端）→ Google 試算表
> ```
> 本機（localStorage）仍是主儲存，離線照記；Google 是額外的雲端備份。

---

## 步驟一：建立試算表

1. 到 [sheets.google.com](https://sheets.google.com) 新建一份空白試算表。
2. 取個名字，例如「旅遊記帳本」。
3. 不用自己排表頭，程式第一次寫入時會自動建分頁與表頭。

## 步驟二：貼上接收程式

1. 在那份試算表上方選單：**擴充功能 → Apps Script**。
2. 會開一個程式編輯器，把裡面預設的 `function myFunction(){}` 全部刪掉。
3. 打開 `Google同步_AppsScript程式碼.gs`，把**整段內容複製貼上**。
4. 按上方 💾（或 Ctrl/Cmd + S）存檔。

## 步驟三：部署成網頁應用程式

1. 編輯器右上 **部署 → 新增部署**。
2. 齒輪選 **網頁應用程式（Web app）**。
3. 設定：
   - 說明：隨意（例：旅遊記帳）
   - **執行身分：我自己**
   - **誰可以存取：所有人**（記帳頁才送得進來）
4. 按 **部署**。
5. 第一次會跳**授權**：選你的 Google 帳號 → 可能出現「Google 尚未驗證這個應用程式」→ 點「進階」→「前往（不安全）」→ 允許。（因為這是你自己寫的私人程式，安全。）
6. 部署完成會給一個**網頁應用程式網址**（`https://script.google.com/macros/s/...../exec`）→ **複製它**。

## 步驟四：填回記帳頁

1. 打開 `_kakeibo_記帳.html`（和 `_warikan_分帳.html`）。
2. 找到最上面這行：
   ```js
   const SHEET_API="";
   ```
3. 把網址貼進去：
   ```js
   const SHEET_API="https://script.google.com/macros/s/...../exec";
   ```
4. 順便確認 `TRIP_NAME`（會變成試算表裡的分頁名，多趟共用一份表時用來區分）。
5. 存檔、push 上 GitHub。

## 步驟五：測試

1. 打開記帳頁，記一筆。
2. 看明細上方的小圓標：**「已同步 ✓」**＝成功；**「同步失敗」**＝網址或權限要再檢查（但該筆已存本機，不會丟）。
3. 回試算表看，該筆應該出現了。

## 驗證小技巧

- 想確認後端活著：直接用瀏覽器打開那個 `.../exec` 網址，看到「旅遊記帳接收服務運作中 ✓」就對了。

## 常見狀況

| 狀況 | 原因 / 處理 |
|---|---|
| 一直「同步失敗」 | SHEET_API 網址貼錯、或部署時「誰可以存取」沒設成「所有人」 |
| 試算表沒出現資料 | 確認用的是 `.../exec` 結尾的網址（不是 `/dev`）；重新部署後網址可能變，要更新 SHEET_API |
| 改了程式碼沒生效 | Apps Script 要**重新部署**（部署 → 管理部署 → 編輯 → 版本選「新版本」）才會更新 |
| 刪除的帳目 | 試算表不會真的刪列，而是把「動作」欄標成「已刪除」，保留軌跡 |

## 隱私提醒

- 「誰可以存取＝所有人」指的是「**知道那串網址的人能送資料進來**」，但網址沒外流就無人知道，一般使用足夠安全。
- **試算表本身**的分享設定另外管：預設只有你能看；要給家人看就把試算表分享給特定 Google 帳號（建議「特定人」而非「知道連結都能編輯」）。

## 多趟共用一份表

- 同一份試算表可服務所有旅程：每趟記帳頁的 `TRIP_NAME` 不同（如 `東京202610`、`大阪202504`），程式會自動為每趟建一個分頁，互不混淆。
- 換趟只需改記帳頁的 `KEY`、`TRIP_NAME`，`SHEET_API` 可共用同一個。
- **旅途日記**也用同一套：它的資料會自動寫進「<帳本>-日記」分頁（與記帳分開），同一個 `SHEET_API` 即可，不用另外設定。

## 讓家人意見也匯進同一份表

家人意見走 Google 表單（不走 Apps Script），但回應可以匯進同一份試算表，集中查看：

1. 開啟你的 Google 表單 → 上方「回應」分頁。
2. 點右上的試算表圖示（建立試算表）。
3. 選「**選取現有試算表**」→ 挑這份「我的旅遊資料」表。
4. 表單回應就會以一個新分頁存進同一份檔。

這樣記帳、分帳、日記（Apps Script）＋ 家人意見（表單）全部落在同一份試算表的不同分頁。詳見規格書第二十三章。
